1 Overview

This document outlines a modernization plan for the SE Lecturer Productivity Hub — a React + TypeScript SPA for university lecturers. The app currently works as a no-build prototype using esm.sh CDN imports. This roadmap defines where we are, where we're going, and exactly how to get there.

💡

Goal: Evolve from "works on my machine" prototype to a well-architected, developer-friendly application that's easy to extend, deploy, and maintain — while preserving all existing functionality and the current visual design.

Priority Matrix

Critical Broken imports, runtime crashes — fix immediately
High Code quality, DRY principles, developer experience
Medium Build tooling, deployment readiness
Low UX polish, tests, nice-to-haves

5 Phase 2 — Architecture & DRY

Eliminate code duplication and establish shared patterns across all 13 features.

Task 2.1   Create useStore() Custom Hook

Problem: Every feature component repeats these 3 lines:

const [data, setData] = useState<AppData | null>(null);
useEffect(() => { getStore().then(setData); }, []);
if (!data) return <div>Loading...</div>;

That's ~40 lines duplicated across 13 components (520+ lines of boilerplate).

What We'll Do

Create a single useStore hook that handles loading state, data fetching, and provides a typed setter.

Files Changed

src/hooks/useStore.ts — NEW
src/utils/store.ts — Remove getStore (keep types + saveStore + initializeStore)
ALL feature components — Replace 3-line pattern with 1-line hook

Before → After

❌ Before (every component)
const [data, setData] = useState<AppData>(null);
const [showModal, setShowModal] = useState(false);
const [editTask, setEditTask] = useState<PersonalTask>(null);
...
useEffect(() => { getStore().then(setData); }, []);
if (!data) return <div>Loading...</div>;
✅ After (every component)
const { data, loading, refresh } = useStore();
const [showModal, setShowModal] = useState(false);
const [editTask, setEditTask] = useState<PersonalTask>(null);
...
if (loading) return <div>Loading...</div>;

Hook Implementation

import { useState, useEffect } from 'react';
import { getStore, saveStore, AppData } from '../utils/store';

export function useStore() {
  const [data, setData] = useState<AppData | null>(null);
  const [loading, setLoading] = useState(true);

  const refresh = useCallback(() => {
    setLoading(true);
    getStore().then(d => { setData(d); setLoading(false); });
  }, []);

  useEffect(() => { refresh(); }, [refresh]);

  const update = useCallback(async (updater: (prev: AppData) => AppData) => {
    const newData = updater(data!);
    setData(newData);
    await saveStore(newData);
  }, [data]);

  return { data, loading, update, refresh };
}

6 Phase 3 — Build Setup

Introduce Vite for a proper development experience. This is the biggest structural change.

Task 3.1   Initialize Vite + React + TypeScript

Why: The current CDN-based approach works but has no hot module replacement, no bundling, no TypeScript compilation, and is hard to deploy. Vite gives us all of this with minimal config.

What We'll Do — Step by Step

  1. Install Vite

    Run npm create vite@latest . -- --template react-ts in the project root (with empty package.json). This scaffolds vite.config.ts, tsconfig.json, and updates index.html.

  2. Remove Import Map

    Delete the <script type="importmap"> block from index.html. Vite handles module resolution natively — no CDN needed.

  3. Update Entry Point

    Rename src/index.tsx → src/main.tsx (Vite convention). Update the <script> tag in index.html to point to /src/main.tsx.

  4. Move Store to Async Compatibility

    The current window.persistentStorage API (OPFS) is browser-specific. Vite doesn't change this, but ensure store.ts still works as-is. The async functions remain unchanged.

  5. Verify All Imports

    Run npx tsc --noEmit to check for TypeScript errors. Fix any issues (mostly just ensuring .tsx extensions are handled by Vite's resolver).

  6. Add npm Scripts

    Add to package.json: "dev": "vite", "build": "tsc && vite build", "preview": "vite preview".

Task 3.2   Add React Router for Deep Linking

Why: Currently, view state lives in App.tsx as useState. You can't bookmark or share a link to a specific view. React Router solves this.

What We'll Do

  1. Install

    npm install react-router-dom

  2. Refactor App.tsx

    Replace the switch(view) pattern with <Routes> and <Route> components.

  3. Update Sidebar.tsx navigation

    Replace onClick={() => onNavigate(view)} with <Link to="/view"> or useNavigate().

  4. Add <Navigate /> for default route

    Redirect / to /dashboard.

Refactored App.tsx

import { BrowserRouter, Routes, Route, Navigate } from 'react-router-dom';
import Sidebar from './components/Sidebar';
import Dashboard from './components/Dashboard';
// ... other imports

export default function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Layout />}>
          <Route index element={<Navigate to="/dashboard" />} />
          <Route path="/dashboard" element={<Dashboard />} />
          <Route path="/timetable" element={<Timetable />} />
          <Route path="/students" element={<Students />} />
          /* ... all other routes ... */
          <Route path="*" element={<Navigate to="/dashboard" />} />
        </Route>
      </Routes>
    </BrowserRouter>
  );
}

function Layout() {
  return (
    <div className="app-shell">
      <Sidebar />
      <main className="main-content">
        <Outlet />
      </main>
    </div>
  );
}

import { describe, it, expect } from 'vitest';
import { calculateStreak } from '../src/utils/journal';

describe('calculateStreak', () => {
  it('returns 0 for empty entries', () => {
    expect(calculateStreak([])).toBe(0);
  });

  it('counts consecutive days', () => {
    const entries = [
      { date: '2025-06-28' },
      { date: '2025-06-27' },
      { date: '2025-06-26' },
    ];
    // Mock new Date() to return 2025-06-28
    vi.setSystemTime(new Date('2025-06-28'));
    expect(calculateStreak(entries)).toBe(3);
  });

  it('breaks streak on gap', () => {
    const entries = [
      { date: '2025-06-28' },
      { date: '2025-06-26' }, // gap on 27th
    ];
    vi.setSystemTime(new Date('2025-06-28'));
    expect(calculateStreak(entries)).toBe(1);
  });
});
